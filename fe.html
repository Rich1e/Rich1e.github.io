IntelliJ IDEA License Server 搭建

作为一枚 FEer，webstorm 应该是必备的开发利器了。它友好支持各个版本的JS语法（ES3/5/6/7），内置的调试工具以及 nodejs 支持都非常完整。

Jetbrains 旗下软件更新挺快，几乎每一个热门 web 框架的集成和调试，都能得到很快的响应。这应该是我比较看重的一个点，其他软件虽然也可以做到，但需要自己手动配置和维护。

下载安装包和使用

v1.5 下载（需要FQ）

原作者教程地址，http://blog.lanyus.com/archives/174.html，如果有不懂的问题，可以在作者blog留言，很多常见问题，在留言区也可以看到解决方案，牛人还是不少。

下载后有很多版本，如果你电脑是 windows，对应的使用 exe 后缀，Mac OS 使用 darwin 后缀。

以下简述一下 Mac OS 的使用流程：

  1. cd 至IntelliJIDEALicenseServer目录
  2. 修改权限，chmod 777 ./IntelliJIDEALicenseServer_darwin_amd64
  3. 查看命令，./IntelliJIDEALicenseServer_darwin_amd64 -h（这个步骤可以忽略，但是对于从事技术的人还是建议看看，有帮助）
  4. 执行命令，./IntelliJIDEALicenseServer_darwin_amd64 -p 端口号 -u 注册名

  5. 在打开的 License Activation 窗口中选择 “License server” ，在输入框输入下面的网址：

    http://127.0.0.1:端口号

    点击：Activate 即可。

本地搭建服务器

此为 Mac 下搭建方式。

  1. 新建一个 com.local.IntelliJIDEALicenseServerDarwinAmd64.plist 文件

    <?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
  <dict>
    <key>KeepAlive</key>
    <false/>
    <key>RunAtLoad</key>
    <true/>
    <key>Label</key>
    <string>com.local.IntelliJIDEALicenseServerDarwinAmd64</string>
    <key>ProgramArguments</key>
    <array>
      <string>Your Path/IntelliJIDEALicenseServer/IntelliJIDEALicenseServer_darwin_amd64</string>
      <string>-l</string>
      <string>0.0.0.0(绑定的IP地址)</string>
      <string>-p</string>
      <string>1081(大于1024的端口号)</string>
      <string>-u</string>
      <string>Your Name(显示被激活的用户名)</string>
    </array>
  </dict>
</plist>

  2. 将文件文件放至 ~/Library/LaunchAgents/ 目录下。

  3. 加载配置好的 .plist 文件:

    launchctl load -w ~/Library/LaunchAgents/com.local.IntelliJIDEALicenseServerDarwinAmd64.plist

launchctl 命令

# 加载一个服务到启动列表
launchctl load -w ~/Library/LaunchAgents/your.plist 
# 卸载一个服务
launchctl unload  ~/Library/LaunchAgents/your.plist 
# 停止
launchctl stop <<Service Name>>
# 开始
launchctl start <<Service Name>>
# 移除
launchctl remove <<Service Name>>
# kill
launchctl kill <<Service Name>>
# 查看所有服务
launchctl list
# 查看服务状态
launchctl list | grep <<Service Name>>

其他一些有用的命令

# 查看服务是否生效
ps aux | grep <<Service Name>>
# 查看 plist 格式是否正确
plutil -lint com.local.IntelliJIDEALicenseServerDarwinAmd64.plist

Fast Configure Of Sublime Text

我是一枚 FE，接触 Sublime Text 有几年时间了，一直在 windows 和 osx 两个平台上使用，感觉非常的棒！这期间有很多心得和总结想要跟大家分享，所以写了此文。

本文欲打造一个最小学习成本又可快速投入使用的入门配置。

倘若你已急不可耐，可以略过其他，根据 Edit Configuration File 直接配置 Sublime Text 。如有兴致也可以细细评味本文其他内容。

Install Package Control

Sublime Text 3

import urllib.request,os,hashlib; h = '2915d1851351e5ee549c20394736b442' + '8bc59f460fa1548d1514676163dafc88'; pf = 'Package Control.sublime-package'; ipp = sublime.installed_packages_path(); urllib.request.install_opener( urllib.request.build_opener( urllib.request.ProxyHandler()) ); by = urllib.request.urlopen( 'http://packagecontrol.io/' + pf.replace(' ', '%20')).read(); dh = hashlib.sha256(by).hexdigest(); print('Error validating download (got %s instead of %s), please try manual install' % (dh, h)) if dh != h else open(os.path.join( ipp, pf), 'wb' ).write(by)

Sublime Text 2

import urllib2,os,hashlib; h = '2915d1851351e5ee549c20394736b442' + '8bc59f460fa1548d1514676163dafc88'; pf = 'Package Control.sublime-package'; ipp = sublime.installed_packages_path(); os.makedirs( ipp ) if not os.path.exists(ipp) else None; urllib2.install_opener( urllib2.build_opener( urllib2.ProxyHandler()) ); by = urllib2.urlopen( 'http://packagecontrol.io/' + pf.replace(' ', '%20')).read(); dh = hashlib.sha256(by).hexdigest(); open( os.path.join( ipp, pf), 'wb' ).write(by) if dh == h else None; print('Error validating download (got %s instead of %s), please try manual install' % (dh, h) if dh != h else 'Please restart Sublime Text to finish installation')

Add Right-Click Menu by Windows

Windows Registry Editor Version 5.00

[HKEY_CLASSES_ROOT\*\shell\Open with Sublime Text 3]
"Icon"="D:\\Program Files\\Sublime Text Build 3083 x64\\sublime_text.exe,0"

[HKEY_CLASSES_ROOT\*\shell\Open with Sublime Text 3\command]
@="D:\\Program Files\\Sublime Text Build 3083 x64\\sublime_text.exe %1"

将以上代码保存为.reg文件，点击执行就可以加入右键菜单了。

（Ps：sublime 安装路径需以实际情况自行修改）

Mac 下 iTerm2 实现 ssh 自动登录

使用 iTerm2 的时候，总是要一遍遍的敲用户名、密码。 我在想， 能不能像 Windows 的软件一样，可以直接让软件记住。然后只要点击一下，就直接 ssh 到远程服务器上面去了。现在可以用 expect 脚本实现。

安装 expect

brew install expect

运行脚本

#!/usr/bin/expect -f
  set user <用户名>
  set host <ip地址>
  set port <端口>
  set password <密码>
  set timeout -1

  spawn ssh $user@$host -p $port
  expect "*assword:*"
  send "$password\r"
  interact
  expect eof

iTerm2 配置

打开 Preferences -> Profiles，然后在 Command 中填入你的脚步路径： ~/your_path

permission denied: ~/your_path

加上可执行权限，方式是 

chmod -R a+x ~/your_path
