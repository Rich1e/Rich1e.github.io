数据类型、原始值和对象

JavaScript 是一种弱类型或者说动态语言。这意味着你不用提前声明变量的类型，在程序运行过程中，类型会被自动确定。这也意味着你可以使用同一个变量保存不同类型的数据：

var foo = 42;    // foo is a Number now
var foo = "bar"; // foo is a String now
var foo = true;  // foo is a Boolean now

最新的 ECMAScript 标准定义了 7 种数据类型:

  • 6 种 原始类型:
    1. Boolean
    2. Null (Null 类型只有一个值： null)
    3. Undefined (一个没有被赋值的变量会有个默认值 undefined)
    4. Number
    5. String
    6. Symbol (ECMAScript 6 新定义)
  • 1 种 对象类型，又叫引用类型:
    1. Object (Array 和 function 是特殊的对象)

原始值

除 Object 以外的所有类型都是不可变的（值本身无法被改变）。例如，与 C 语言不同，JavaScript 中字符串是不可变的（译注：如，JavaScript 中对字符串的操作一定返回了一个新字符串，原始字符串并没有被改变）。我们称这些类型的值为“原始值”。

对象

在 Javascript 里，对象可以被看作是一组属性的集合。用对象字面量语法来定义一个对象时，会自动初始化一组属性。（也就是说，你定义一个 var a = "Hello"，那么 a 本身就会有 a.substring 这个方法，以及 a.length 这个属性，以及其它；如果你定义了一个对象，var a = {}，那么a就会自动有 a.hasOwnProperty 及 a.constructor 等属性和方法。）而后，这些属性还可以被增减。属性的值可以是任意类型，包括具有复杂数据结构的对象。属性使用键来标识，它的键值可以是一个字符串或者符号值（Symbol）。

ECMAScript 定义的对象中有两种属性：数据属性和访问器属性。

以上都是概念，实际使用时，我们该怎么理解呢？上一份代码。

var a = 'a', b = a;
b = 'b'
console.log(a); // 'a'
console.log(b); // 'b'

var red = {color: 'red'}, blue = red;
blue.color = 'blue';
console.log(red.color); // blue
console.log(blue.color); // blue
console.log(red === blue); // true
console.log(blue === {color: 'blue'}); // false

原始值是不可更改的：任何方法都无法更改（或“突变”）一个原始值（你无法修改值本身，你只能给代表它的变量重新赋值，将原来的值覆盖）。

即使两个引用值包含同样的属性及相同的值，它们也是不相等的。对象的比较均是引用的比较：当且仅当它们引用同一个基对象时，它们才相等。并且，修改引用值时，基对象也会被修改。

如果你想得到一个对象或数组的副本，则必须显式复制对象的每个属性或数组的每个元素，可以使用这个函数：

function clone(p) {
  var c = {};

  for (var i in p) {
    c[i] = p[i];
  }　　　
  c.uber = p;
  return c;
}

// 加强版
function deepClone(p, c) {
  var c = c || {};
  for (var i in p) {
    if (typeof p[i] === 'object') {
      c[i] = (p[i].constructor === Array) ? [] : {};
      deepClone(p[i], c[i]);
    } else {
      c[i] = p[i];
    }
  }
  return c;
}

  • 捕获型事件(Event Capturing)

  • DOM标准的事件

变量提升

变量的提升只是其定义提升，而变量的赋值并不会提升。

变量的几种形态：

  1. 语言自定义变量：所有的作用域中都存在 this 和 arguments 这两个默认变量
  2. 函数形参：函数的形参存在函数作用域中
  3. 函数声明：function foo() {}
  4. 变量定义：var foo

变量的影响范围：函数级作用域（function-level scope）。在 es6 之前，javascript 中没有块级作用域，但可以通过闭包来实现类似效果。

下面有两个例子：

  • 第一行：声明了一个局部变量 foo, 并赋值
  • 第二行：声明了一个函数 bar，在内部声明了一个新函数 foo，并赋予了 foo 新值

  • 第一行：声明了一个局部变量 foo, 并赋值
  • 第二行：声明了一个函数 bar，当 if 语句为真时，重新声明了一个局部变量 foo，并赋予新值，最后输出 foo

怎么理解呢？上面的代码好比是这样：

var foo = 1;
function bar() {
    var foo;
    if (!foo) {
        foo = 10;
    }
    console.log(foo);
}
bar();

var foo = 1;
function bar() {
    function foo() {}
    foo = 10;
    return;
}
bar();
console.log(foo);

这就是为什么，我们写代码的时候，变量定义总要写在最前面。

作用域和原型链

  • 第二行：声明一个函数 test，赋予了 scope 新值，并显示声明了一个全局变量 myscope，然后返回这两个值。
